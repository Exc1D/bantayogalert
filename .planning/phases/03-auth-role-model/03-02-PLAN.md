**Auth flows:**
- Login: `signInWithEmailAndPassword(auth, email, password)` or `signInWithPopup(auth, googleProvider)`
- Register: `createUserWithEmailAndPassword(auth, email, password)` then `sendEmailVerification(user)`
- After register, user is logged in but must verify email before the account becomes fully usable (Firebase default behavior)
- Google OAuth: `signInWithPopup(auth, googleProvider)` per D-44

**Route protection (D-49, D-50):**
- `/app/*` requires authentication (redirect to `/auth/login`)
- `/admin/*` requires `municipal_admin` or `provincial_superadmin` role
- React Router loader functions check auth state per D-49

**Notification preferences UI:**
- Simple toggles for `pushEnabled`, `emailEnabled`
- Multi-select for `alertTypes` (flood, landslide, fire, earthquake, medical, all)

**Note:** This plan covers UI pages and route guards only. The setUserRole callable and custom claims Cloud Functions are in plan 03-03.
</context>

  <files>src/lib/auth/hooks.ts, src/lib/auth/operations.ts</files>
  <read_first>
    src/lib/auth/AuthProvider.tsx
    src/types/user.ts
  </read_first>
  <action>
    Create `src/lib/auth/hooks.ts`:
    - Export `useAuth()` hook that uses `useContext(AuthContext)` and returns `{ user, customClaims, isLoading, isAuthenticated }`
    - Export `useRequireAuth()` hook that:
      - Uses `useAuth()` to get auth state
      - If `!isLoading && !isAuthenticated`, calls `navigate('/auth/login')`
      - If authenticated, returns auth state
      - Returns `null` during loading to prevent flash
    - Export `useRequireRole(allowedRoles: UserRole[])` hook that:
      - Uses `useAuth()` to get customClaims
      - Redirects to `/app` if authenticated but role not in `allowedRoles`
      - Returns the custom claims if authorized

    Create `src/lib/auth/operations.ts`:
    - Export `loginWithEmail(email: string, password: string)` using `signInWithEmailAndPassword(auth, email, password)`
    - Export `registerWithEmail(email: string, password: string, displayName: string)` that:
      1. Calls `createUserWithEmailAndPassword(auth, email, password)`
      2. Calls `updateProfile(user, { displayName })`
      3. Calls `sendEmailVerification(user)`
      4. Creates user document in Firestore at `/users/{userId}` with default claims
    - Export `loginWithGoogle()` using `signInWithPopup(auth, googleProvider)`
    - Export `logout()` using `signOut(auth)`
    - Export `updateUserProfile(data: { displayName?: string; notificationPreferences?: NotificationPreferences })` that updates Firestore user document

    **Firestore user document on registration:**
    ```typescript
    await setDoc(doc(db, 'users', user.uid), {
      email: user.email,
      displayName: displayName,
      role: 'citizen',
      provinceCode: 'CMN',
      municipalityCode: null,
      notificationPreferences: {
        pushEnabled: false,
        emailEnabled: true,
        alertTypes: ['all'],
      },
      createdAt: serverTimestamp(),
      updatedAt: serverTimestamp(),
    })
    ```
  </action>

    Create `src/lib/router/guards.tsx`:
    - Export `ProtectedRoute` component:
      - Props: `children: React.ReactNode`
      - Uses `useRequireAuth()` hook
      - If loading: renders nothing (or spinner)
      - If unauthenticated: redirects to `/auth/login`
      - If authenticated: renders children
    - Export `AdminRoute` component:
      - Props: `children: React.ReactNode`
      - Uses `useRequireRole([UserRole.MunicipalAdmin, UserRole.ProvincialSuperadmin])`
      - If unauthorized (wrong role): redirects to `/app`
      - If authorized: renders children

    Update `src/App.tsx` routes:
    - Add `/auth/login` route pointing to `src/app/auth/login/page.tsx`

    - Add `/app/*` routes wrapped in `ProtectedRoute`
    - Add `/admin/*` routes wrapped in `AdminRoute`
