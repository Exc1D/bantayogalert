- D-46: setUserRole callable CF - superadmin only, sets role + municipalityCode + provinceCode on user record

- D-48: Default role on registration: citizen with provinceCode='CMN', municipalityCode=null

**UserRole enum (from Phase 2):**
- `citizen` = 'citizen'
- `municipal_admin` = 'municipal_admin'
- `provincial_superadmin` = 'provincial_superadmin'

**Province code:** Always 'CMN' for Camarines Norte

**Atomic claims update pattern (D-47):**
1. Update Firestore user document
2. Call admin.auth().setCustomUserClaims()
3. Return success only if both succeed

  <action>
    Create `functions/src/auth/setUserRole.ts`:
    - Import `functions` from `firebase-functions`
    - Import `setCustomClaims, isSuperadmin` from `./claims`
    - Import `getFirestore` from `firebase-admin/firestore`

    Export `setUserRole` as `onCall` callable:
    - Validate callable auth context: `context.auth` must exist
    - Get caller's custom claims from `context.auth.token`
    - Verify caller is `provincial_superadmin` using `isSuperadmin(context.auth.token)`
    - If not superadmin, throw `functions.https.HttpsError('permission-denied', 'Only superadmins can assign roles')`
    - Parse request data: `{ uid: string; role: string; municipalityCode: string | null }`
    - Validate required fields: `uid`, `role`
    - Validate `role` is one of: `citizen`, `municipal_admin`, `provincial_superadmin`
    - Validate `provinceCode` is always `'CMN'`
    - For `municipal_admin` role, `municipalityCode` MUST be provided and valid (3-4 chars)
    - For `citizen` role, `municipalityCode` MUST be `null`
    - For `provincial_superadmin` role, `municipalityCode` MUST be `null`
    - Call `setCustomClaims(uid, { role, municipalityCode, provinceCode: 'CMN' })`
    - Return `{ success: true, role, municipalityCode, provinceCode: 'CMN' }`

    **Error handling:**
    - Missing uid: `functions.https.HttpsError('invalid-argument', 'uid is required')`
    - Invalid role: `functions.https.HttpsError('invalid-argument', 'Invalid role')`
    - Missing municipalityCode for municipal_admin: `functions.https.HttpsError('invalid-argument', 'municipalityCode required for municipal_admin')`
    - Non-null municipalityCode for citizen/superadmin: `functions.https.HttpsError('invalid-argument', 'municipalityCode must be null for citizen and superadmin')`

    **Input validation:**
    ```typescript
    const VALID_ROLES = ['citizen', 'municipal_admin', 'provincial_superadmin']
    const MUNICIPALITY_CODE_REGEX = /^[A-Z]{3,4}$/ // 3-4 uppercase letters
    ```
  </action>

    - Calls setCustomClaims to update both Firestore doc and ID token
