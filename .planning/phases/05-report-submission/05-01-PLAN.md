<task type="auto">
  <name>Task 2: Create submitReport Cloud Function</name>
  <files>functions/src/reports/submitReport.ts</files>
  <action>
Create `functions/src/reports/submitReport.ts` implementing the `submitReport` callable CF.

**Input validation (Zod schema):**
```typescript
const SubmitReportDataSchema = z.object({
  type: z.nativeEnum(IncidentType),
  severity: z.nativeEnum(Severity),
  description: z.string().min(10).max(2000),
  municipalityCode: z.string().min(3).max(4),
  barangayCode: z.string().min(6).max(7),
  exactLocation: z.object({
    lat: z.number().min(13.8).max(14.8),
    lng: z.number().min(122.3).max(123.3),
  }),
  mediaUrls: z.array(z.string().url()).max(5).default([]),
})
```

**Implementation:**
1. `validateAuthenticated(auth)` — throws if not authenticated (from security module)
2. Validate role is citizen using `validateRole(auth.token, UserRole.Citizen)`
3. Parse and validate data with SubmitReportDataSchema
4. Rate limit check via `checkRateLimit(userId)` — throws if exceeded
5. Compute geohash via `ngeohash.encode(exactLocation.lat, exactLocation.lng, 9)`
6. Generate `reportId = db.collection('reports').doc().id` BEFORE transaction
7. Run `db.runTransaction` creating three docs atomically:
   - `reports/{reportId}`: public doc with geohash, workflowState=pending, verified=false
   - `report_private/{reportId}`: exactLocation, reporterEmail, reporterName, ownerStatus=submitted, activityLog entry
   - `report_ops/{reportId}`: empty initial doc { id: reportId }
8. Increment rate limit counter via `incrementRateLimit(userId)`
9. Return `{ reportId }`

**Coordinate bounds:** lat 13.8-14.8, lng 122.3-123.3 (already in Zod schema)

**Import ngeohash in functions:**
```bash
cd functions && npm install ngeohash@0.6.3
```
