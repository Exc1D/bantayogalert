Purpose: Users can navigate the app and open panel content via role-appropriate nav items.
Output: DesktopNavRail.tsx, MobileBottomTabs.tsx, NavItem.tsx
</objective>


  <name>Task 1: Create NavItem reusable component</name>
  <files>src/app/shell/NavItem.tsx</files>
  <action>
    Create `src/app/shell/NavItem.tsx` — a reusable nav item for both desktop rail and mobile tabs.

    Props interface:
    ```typescript
    interface NavItemProps {
      icon: LucideIcon
      label: string
      isActive?: boolean
      onClick: () => void
      badge?: number  // optional notification count
      variant: 'rail' | 'tab'  // 'rail' = vertical (desktop), 'tab' = horizontal (mobile)
    }
    ```

    Implementation:
    - Uses the passed `icon` component directly (already a Lucide icon component)
    - **Rail variant**: vertical layout, icon on top (24px), label below (10px text), full width
      - Default: `text-gray-400`, bg transparent
      - Active: `text-primary-600 dark:text-primary-400`, `bg-primary-50 dark:bg-primary-900/20`, left border indicator `border-l-2 border-primary-600`
      - Hover: `text-gray-700 dark:text-gray-200`, `bg-gray-100 dark:bg-gray-800`
    - **Tab variant**: horizontal layout, icon (24px) above label (10px), equal flex distribution
      - Default: `text-gray-500`
      - Active: `text-primary-600 dark:text-primary-400`
      - Hover: `text-gray-700 dark:text-gray-200`
    - `onClick` is passed directly — parent decides what action to take (open drawer, switch tab, navigate)
    - If `badge` is provided and > 0, show a red dot or count badge on the icon (top-right corner)

    Tailwind: `flex flex-col items-center justify-center gap-0.5 cursor-pointer transition-colors rounded-lg`
  </action>
