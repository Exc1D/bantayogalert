## VALID_TRANSITIONS reference

```typescript
const VALID_TRANSITIONS: Record<WorkflowState, WorkflowState[]> = {
  [WorkflowState.Pending]: [WorkflowState.Verified, WorkflowState.Rejected],
  [WorkflowState.Verified]: [WorkflowState.Dispatched, WorkflowState.Rejected],
  [WorkflowState.Dispatched]: [WorkflowState.Acknowledged, WorkflowState.Dispatched], // reroute
  [WorkflowState.Acknowledged]: [WorkflowState.InProgress],
  [WorkflowState.InProgress]: [WorkflowState.Resolved],
  [WorkflowState.Rejected]: [], // terminal
  [WorkflowState.Resolved]: [], // terminal
}
```

## Action Buttons Per State (derived from VALID_TRANSITIONS)

- **pending**: [Verify] [Reject] buttons visible
- **verified**: [Dispatch] button visible
- **dispatched**: [Acknowledge] [Reroute] buttons visible
- **acknowledged**: [Mark In Progress] button visible
- **in_progress**: [Resolve] button visible
- **rejected**: no action buttons (terminal)
- **resolved**: no action buttons (terminal)

## Reject Flow (D-167)

Reject opens a confirmation dialog with:
- Reason text field (required, min 1 char)
- Category dropdown: insufficient_info / duplicate / out_of_area / false_report / other (optional)

After confirm: call triageReject({ reportId, expectedVersion, reason, category })

## Dispatch Flow (D-169-D-173)

Clicking Dispatch opens ContactPickerModal.
After selecting a contact: call triageDispatch({ reportId, expectedVersion, contactId, routingDestination?, dispatchNotes? })

## Priority & Notes (always available)

PriorityStars always visible in the panel header area.
Internal notes textarea always visible (admin-only).
Save buttons call triageUpdatePriority and triageUpdateNotes respectively.
