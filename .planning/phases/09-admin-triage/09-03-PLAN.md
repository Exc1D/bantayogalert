## triageReroute special case

- D-170 says reroute can be done on "dispatched or in_progress" reports
- VALID_TRANSITIONS['dispatched'] includes 'dispatched' (self-loop for reroute)
- VALID_TRANSITIONS['in_progress'] does NOT include any reroute transition
- But D-170 says in_progress can be rerouted — this means we need to validate reroute specially
- Actually: reroute doesn't change workflowState, it only updates assignedContactSnapshot + dispatchNotes + routingDestination
- So triageReroute should NOT validate against VALID_TRANSITIONS — it updates the SAME state with new contact
- However, TRI-08 says "Reroute a dispatched or in_progress report to a different contact" — this means only these two states allow reroute
- Check currentState === 'dispatched' || currentState === 'in_progress' to allow reroute

    4. Return { success: true }

    Note: This is the only triage CF that does NOT update the reports/{id} document.
    It only updates assignedContactSnapshot and related fields on report_private/report_ops.
